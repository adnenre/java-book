# Historique des versions de Java et fonctionnalités

Java a évolué à travers de nombreuses versions. Depuis Java 9, Oracle a adopté un **modèle de publication basé sur le temps** avec une nouvelle version tous les six mois. Les versions Long‑Term Support (LTS) bénéficient de mises à jour pendant plusieurs années.

## Cadence de publication

- **Versions non‑LTS** : tous les six mois (mars et septembre) – supportées uniquement jusqu’à la version suivante.
- **Versions LTS** : tous les quelques années (Java 8, 11, 17, 21, 25, …) – reçoivent des mises à jour pendant au moins 3 ans (Oracle) ou plus (distributions OpenJDK).

## Choisir une version

Pour les nouveaux projets, **Java 21 ou 25 (LTS)** sont recommandés car ils sont stables et supportés à long terme. Vous pouvez aussi utiliser la dernière version (Java 26) pour expérimenter de nouvelles fonctionnalités comme le support HTTP/3 et les capacités IA améliorées.

## Rétrocompatibilité

Java s’engage fortement pour la rétrocompatibilité. Un code écrit pour Java 1.2 s’exécutera généralement sur Java 26, à de rares exceptions près pour les fonctionnalités dépréciées supprimées après de longs cycles de dépréciation.
